# receiver/ — the uniform receiver daemon

You are in the receiver daemon: the long-running process that connects a hosted brain to the coordinator and executes delegated tasks. Read `../CLAUDE.md` (connector) and the root `../../CLAUDE.md` first.

## Shape
- **`core/`** — brain-agnostic engine: `config.js` (env/config load), `contract.js` (the wire contract: task/result/`ask`/session-ref messages), `transport.js` (WebSocket transport + reconnect), `receiver.js` (the loop that ties them together).
- **`adapters/<brain>/`** — per-brain glue. `claude/` and `openclaw/` (each an `index.js`); `adapters/index.js` selects by brain. An adapter translates a delegated task into a real turn on that brain and the result back into the contract.
- **`ws_client.js`** — the daemon entrypoint (`node receiver/ws_client.js`).
- **`ammunity-receiver.service`** — a placeholder TEMPLATE only. The real per-host unit is GENERATED by the installer (Batch D). **Never commit a filled-in unit** (the 2026-06-29 leak).

## Rules
- **Newest contract only — no back-compat shims.** Design principle #7: no contract change ships until a live agent comprehends it. Step 3 (`receiver_session_ref` carry on `ask`/resume) is closed end-to-end as of 2026-06-29b.
- New adapters must satisfy the same `core/contract.js` so the daemon stays brain-agnostic. Codex adapter is deferred to Tier-2.
- The WebSocket transport must survive 1006 drops (validated: recovers in ~1–2s). Don't regress the reconnect/watchdog logic in `transport.js`.
- Syntax-check touched files with `node --check`. Run `npm test` (`tests/test_receiver.mjs`) before pushing.
